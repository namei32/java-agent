# 运行手册命令路径无关化设计

- 状态：已实现并验证
- 日期：2026-07-13
- 范围：`docs/runbooks/local-development.md`

## 1. 问题

从编辑器直接打开运行手册并启动终端时，当前目录可能是 `docs/runbooks`。现有命令使用 `.env`、`./mvnw`、`./workspace-java` 和 `agent-bootstrap/target/...` 等相对路径，因此会错误地相对于运行手册目录解析。

## 2. 目标

让每个依赖仓库路径的命令块都能从仓库内任意子目录独立执行，不依赖用户先手工切换到项目根目录，也不硬编码本机绝对路径。

## 3. 方案比较

1. **动态定位 Git 根目录（采用）**：每个相关命令块先执行 `cd "$(git rev-parse --show-toplevel)"`。命令块可独立运行，仓库移动或换电脑后仍有效。
2. **硬编码绝对路径**：最直观，但只适用于当前电脑和当前目录，不能提交为通用文档。
3. **只在文档开头切换一次目录**：内容较短，但后续命令块依赖用户已经执行前置步骤，直接点击单个命令块仍会失败。

## 4. 文档变更

- 在环境准备、Workspace 创建、构建启动、测试 Profile 和缺少配置排查等所有依赖项目文件的命令块中，先动态切换到 Git 根目录。
- 明确说明命令需要从本仓库内部启动；不在 Git 仓库中时，`git rev-parse --show-toplevel` 会失败。
- 保持 Curl 等不读取仓库文件的命令独立，不增加无意义的目录切换。
- 保持现有 API、配置键、运行流程和安全约束不变。

## 5. 验收标准

1. 从项目根目录执行相关命令时行为不变。
2. 从 `docs/runbooks` 执行任意依赖项目路径的命令块时，`.env`、Maven Wrapper、Workspace 和 Boot JAR 均相对于项目根目录解析。
3. 文档不包含 `/Users/namei/...` 等机器专属路径。
4. 每个相关命令块都是自包含的，不依赖之前的命令块改变 Shell 当前目录。
5. Spotless 文档格式检查通过。

## 6. 验证记录

2026-07-13 从 `docs/runbooks` 启动 Shell，动态解析得到项目根目录，并确认 `.env.example`、Maven Wrapper 和 `agent-bootstrap` 均能从根目录正确定位。`./mvnw spotless:check` 通过。
